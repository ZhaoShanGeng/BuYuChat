# 步语 BuYu — 软件规格说明

**版本：** 0.1
**阶段：** 当前实现
**最后更新：** 2026-04-08

本文描述当前代码已经实现的软件能力与边界，作为产品层面的事实说明。

## 1. 产品定义

BuYu 是一个本地优先的 AI 聊天客户端。

当前产品形态：

- 前端：Svelte 5 单页工作台
- 客户端壳：Tauri v2
- 后端：Rust 本地服务层
- 数据存储：SQLite

当前产品目标：

1. 让用户在本地创建和管理多轮 AI 会话。
2. 让用户配置多个 AI 渠道、模型和 Agent。
3. 让用户查看流式生成过程，并保留完整会话历史。
4. 让用户在本地导入、导出和维护自己的配置数据。

## 2. 当前使用场景

当前已经支持的核心使用场景：

1. 创建一个会话并发送消息。
2. 为会话绑定 Agent、渠道和模型。
3. 在生成过程中实时查看回复内容和思考内容。
4. 对历史消息执行重试、编辑、取消、版本切换和版本删除。
5. 管理渠道和模型，并测试渠道连通性。
6. 为会话启用内置工具并查看工具调用结果。
7. 导入导出渠道与模型配置，打开数据目录和日志目录。

## 3. 功能范围

### 3.1 工作台

当前工作台包含 3 个主区域：

- `chat`：会话列表、聊天区、消息时间线、输入区
- `agents`：Agent 列表与 Agent 编辑区
- `settings`：渠道列表、渠道编辑器、模型管理、实用工具区

当前支持桌面端和移动端基础适配：

- 桌面端使用固定左侧图标栏和上下文侧边栏
- 移动端使用抽屉式侧边栏和简化头部操作

### 3.2 会话与消息

当前会话能力：

- 创建会话
- 删除会话
- 重命名会话
- 置顶 / 取消置顶
- 归档 / 取消归档
- 绑定 Agent
- 绑定渠道
- 绑定模型
- 配置会话级启用工具列表

当前消息能力：

- 发送文本消息
- 发送图片附件
- 发送文件附件
- dry run 查看最终 prompt
- 流式生成 assistant 回复
- 取消生成
- assistant reroll
- user reroll
- 编辑消息并可选重新触发后续生成
- 删除版本
- 切换 active version
- 按需加载非 active version 的正文

### 3.3 Agent

当前 Agent 能力：

- 创建 Agent
- 查看 Agent 列表
- 查看单个 Agent
- 更新 Agent 名称和系统提示词
- 启用 / 禁用 Agent
- 删除 Agent

### 3.4 渠道与模型

当前渠道能力：

- 创建渠道
- 查看渠道列表
- 查看单个渠道
- 更新渠道
- 删除渠道
- 测试渠道连通性

当前渠道配置项包括：

- `base_url`
- `api_key`
- `api_keys`
- `auth_type`
- `models_endpoint`
- `chat_endpoint`
- `stream_endpoint`
- `thinking_tags`
- `enabled`

当前模型能力：

- 查看某个渠道下的模型列表
- 创建模型
- 更新模型
- 删除模型
- 从远程渠道拉取模型候选
- 导入远程模型到本地模型表

### 3.5 工具能力

当前产品已经接入内置工具能力。

当前可见的产品级工具能力：

- 获取内置工具列表
- 在会话级启用工具
- 在消息中展示工具调用和工具结果

当前默认内置工具：

- `fetch`

### 3.6 配置维护

当前设置实用工具能力：

- 导出渠道与模型配置到 JSON 文件
- 从 JSON 文件导入渠道与模型配置
- 打开应用数据目录
- 打开应用日志目录

## 4. 数据要求

当前数据要求：

- 所有核心业务数据保存在本地 SQLite
- 会话、消息、版本、渠道、模型、Agent 都必须可重启后恢复
- 流式生成中的正文和 thinking 必须按块持久化
- 配置导入导出文件必须是 JSON

## 5. 交互要求

当前交互要求：

- 生成中的消息必须实时增量显示
- 失败、取消、空回滚必须在界面上可见
- 会话切换后必须能恢复消息列表和 active version
- 设置修改后，工作台需要刷新相关状态
- 移动端必须能完成会话切换、聊天和设置管理的基本流程

## 6. 错误处理要求

当前错误处理要求：

- 后端统一返回结构化错误
- 前端把错误码转换为可读提示
- 生成失败时保留失败版本，不直接丢失上下文
- 渠道测试失败时展示失败原因
- 配置导入失败时给出明确的文件格式提示

## 7. 当前非目标

以下内容当前不属于已实现的软件规格：

- 用户账号体系
- 云端同步
- 多人协作
- 系统钥匙串存储 API Key
- 完整插件市场
- 桌面端窗口级 E2E 测试体系

## 8. 验收口径

当前代码满足软件规格的最低验收口径：

1. 应用可以正常启动并进入工作台。
2. 用户可以配置渠道和模型，并完成渠道连通性测试。
3. 用户可以创建会话、发送消息并收到流式回复。
4. 用户可以对消息执行取消、重试、编辑、切版本和删版本。
5. 用户可以导入导出配置，并在重启后保留本地数据。
